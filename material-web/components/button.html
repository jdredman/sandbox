Usage Guidelines

✅ Do

  • Use filled buttons for primary actions like "Submit" or "Continue"
  • Use outlined buttons for secondary actions like "Cancel" or "Back"
  • Keep button text concise and action-oriented
  • Use sentence case for button labels
  • Provide adequate spacing between buttons

❌ Don't

  • Use multiple filled buttons in the same context
  • Make button text too long or verbose
  • Use ALL CAPS for button text
  • Place buttons too close together without spacing
  • Use icons without clear meaning

Accessibility

Keyboard Navigation

  • Tab - Focus next button
  • Shift + Tab - Focus previous button
  • Enter or Space - Activate button

Screen Readers

  • Buttons have proper ARIA roles
  • Disabled state is announced
  • Icon meanings are conveyed via aria-label

Color Contrast

  • Text meets WCAG AA standards
  • Focus indicators are clearly visible
  • Colors work in high contrast mode
