Usage Guidelines

‚úÖ Do

  • Use consistent spacing tokens throughout your designs
  • Increase spacing to show hierarchy and separation
  • Use smaller spacing for related elements
  • Consider touch targets when spacing interactive elements
  • Test spacing on different screen sizes

‚ùå Don't

  • Use arbitrary spacing values outside the scale
  • Make spacing too tight for comfortable interaction
  • Use the same spacing for all elements regardless of hierarchy
  • Ignore spacing on mobile devices
  • Create inconsistent spacing patterns

Hierarchy Principles

  • 4XL (96px): Major page sections
  • 3XL (64px): Section boundaries
  • 2XL (48px): Content groups
  • XL (32px): Component boundaries
  • LG (24px): Internal component spacing
  • MD (16px): Related elements
  • SM (8px): Closely related items
  • XS (4px): Micro adjustments

Responsive Considerations

  • Reduce spacing on smaller screens
  • Maintain minimum touch target sizes (44px)
  • Use relative units for scalable spacing
  • Test on various device sizes
  • Consider content density needs

Implementation

CSS Custom Properties

Define spacing tokens as CSS custom properties for consistency and easy maintenance.

/* Define spacing tokens */ :root { --spacing-xs: 4px; --spacing-sm: 8px; --spacing-md: 16px; --spacing-lg: 24px; --spacing-xl: 32px; --spacing-2xl: 48px; --spacing-3xl: 64px; --spacing-4xl: 96px; } /* Usage examples */ .card { padding: var(--spacing-lg); margin-bottom: var(--spacing-xl); } .button-group { display: flex; gap: var(--spacing-sm); } .section { margin-bottom: var(--spacing-2xl); } .page-content { padding: var(--spacing-4xl) var(--spacing-xl); }
