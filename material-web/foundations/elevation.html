Elevation

Material Design elevation system for creating depth, hierarchy, and focus in user interfaces.

Elevation Scale

Elevation uses shadows to create depth and hierarchy. Higher elevations appear closer to the user and cast larger, softer shadows.

0
Surface
Background surfaces, no shadow
1
Raised
Cards, buttons at rest
2
Hover
Cards on hover, raised buttons
3
Focus
FABs, active states
4
Overlay
Menus, tooltips, dropdowns
5
Modal
Dialogs, modals, bottom sheets

Usage Guidelines

‚úÖ Do

  • Use elevation to show hierarchy and importance
  • Increase elevation for interactive states (hover, focus)
  • Keep elevation consistent across similar components
  • Use higher elevation for temporary surfaces (modals, menus)
  • Apply elevation meaningfully, not decoratively

‚ùå Don't

  • Use too many different elevation levels
  • Apply heavy shadows to small elements
  • Make permanent surfaces higher than temporary ones
  • Use elevation inconsistently across your interface
  • Overuse high elevation levels

Hierarchy Principles

  • Level 0: Background surfaces
  • Level 1: Base components (cards, buttons)
  • Level 2: Hover states, raised elements
  • Level 3: Focus states, floating action buttons
  • Level 4: Overlay content (menus, tooltips)
  • Level 5: Modal content (dialogs, sheets)

Accessibility

  • Elevation helps users understand interface hierarchy
  • Higher elevation indicates higher importance or focus
  • Don't rely on elevation alone to convey information
  • Ensure sufficient contrast between elevated surfaces
  • Test shadows with high contrast mode

Implementation

CSS Shadow Tokens

Define elevation as CSS custom properties for consistent application across components.

/* Elevation shadow tokens */ :root { --elevation-0: none; --elevation-1: 0 1px 3px rgba(0, 0, 0, 0.12), 0 1px 2px rgba(0, 0, 0, 0.24); --elevation-2: 0 3px 6px rgba(0, 0, 0, 0.16), 0 3px 6px rgba(0, 0, 0, 0.23); --elevation-3: 0 10px 20px rgba(0, 0, 0, 0.19), 0 6px 6px rgba(0, 0, 0, 0.23); --elevation-4: 0 14px 28px rgba(0, 0, 0, 0.25), 0 10px 10px rgba(0, 0, 0, 0.22); --elevation-5: 0 19px 38px rgba(0, 0, 0, 0.30), 0 15px 12px rgba(0, 0, 0, 0.22); } /* Usage examples */ .card { box-shadow: var(--elevation-1); transition: box-shadow 300ms cubic-bezier(0.4, 0.0, 0.2, 1); } .card:hover { box-shadow: var(--elevation-2); } .fab { box-shadow: var(--elevation-3); } .modal { box-shadow: var(--elevation-5); }

Interactive Transitions

/* Smooth elevation transitions */ .interactive-element { box-shadow: var(--elevation-1); transition: box-shadow 300ms cubic-bezier(0.4, 0.0, 0.2, 1), transform 300ms cubic-bezier(0.4, 0.0, 0.2, 1); } .interactive-element:hover { box-shadow: var(--elevation-3); transform: translateY(-2px); } .interactive-element:active { box-shadow: var(--elevation-1); transform: translateY(0); } /* Focus states for accessibility */ .interactive-element:focus-visible { box-shadow: var(--elevation-2), 0 0 0 3px var(--md-sys-color-primary); }

Utility Classes

/* Elevation utility classes */ .elevation-0 { box-shadow: var(--elevation-0); } .elevation-1 { box-shadow: var(--elevation-1); } .elevation-2 { box-shadow: var(--elevation-2); } .elevation-3 { box-shadow: var(--elevation-3); } .elevation-4 { box-shadow: var(--elevation-4); } .elevation-5 { box-shadow: var(--elevation-5); } /* Hover elevation modifiers */ .hover-elevation-1:hover { box-shadow: var(--elevation-1); } .hover-elevation-2:hover { box-shadow: var(--elevation-2); } .hover-elevation-3:hover { box-shadow: var(--elevation-3); }
